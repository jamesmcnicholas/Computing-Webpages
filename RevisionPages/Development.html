Stages of Compilation

Stages of compilation

    The 7 stages in which compilation occurs

  • Lexical Analysis
  • Syntax Analysis
  • Semantic Analysis
  • Intermediate Code Generation
  • Intermediate Code Optmisation
  • Machine code generation
  • Machine code optimisation

Source code analysis

Source code is found on the hard drive. It is moved into the compiler, all the unnecessary comments of whitespace is removed.Next comes the optimisation, in which long or complicated procedures are broken down into their fundamental parts. The code is then analysed and any errors found in the form of Syntax, Semantic, or Lexical are detected, and will stop compilation. Once there are no more errors, the code is then compiled.

    Lexical Analysis

  • Source code is stripped of anything unnecessary (e.g. whitespace and comments)
  • Code is divided into lexemes (Smallest possible pieces of code)
  • Lexical analyser tries to assign a token to each lexeme.
  • Tokens include: Identifiers, Keywords, Operators, Literals & Symbols
  • The end result is a list of lexeme:token pairs
  • If a lexeme cannot be assigned a token, an error is raised

    • Syntax Analysis

  • The stream of generated tokens from the previous process is analysed
  • The tokens are checked against the rules of the language
  • Any failures raise an error, e.g. two different data types in one declaration
  • Tree based data structures are often used in this process

    • Semantic Analysis

  • The code is now checked for logical errors
  • Examples include type mismatch, e.g. assigning an integer to a String
  • Undeclared variables or trying to access a variable out of scope
  • Multiple declarations of the same variable in the same scope
  • Accessing an array with an index out of bounds
  • As before, an error is returned if a rule is broken
  • Not all semantic errors can be caught during compilation
  • Run time errors such as out of bounds integer variable will not be prevented by this step

Code Generation and Optimisation

After the analysis stages are complete, the code moves on to the next step. Here, it is translated into an intermediate code. In languages such as java, this is the final step, as bytecode is what's run in the JVM. In other languages, such as C++, this stage is just another stage in the compilation. Intermediate code is machine independant, which means it can be run on any architecture. Though it has been translated, the intermediary code is not machine code yet.

The code is optimised so that it can run as efficiently as possible a d therefore require less resources, without altering the function as intended. Examples include removing useless variables and simplifying loops.

Final Compilation

At this point, it is determined whether or not the program will include libraries within the compiled file. If so, the static linker will fetch the static libraries and add them to the program loader. If the libraries will instead be dynamic, this step will be skipped for now. Once in the program loader, the dynamic linker will add .dll files to the program. These libraries are dynamic, so are not included with the compiled program, This means they must be present when the file is executed for the program to fnunction correctly. Once this process has finished, the loader loads the file into RAM.
