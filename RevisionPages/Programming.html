Object Oriented Programming - Fundamentals

Many modern languages are based around OOP, including Java, C++, Python & more

The basics are as followed:

    The five basic OOP principles

  1. Code is divided up into objects which contain variables (referred to as attributes) and functions (known as methods)
  2. Objects are made from "blueprints" called classes. Objects are an instance of a class
  3. Objects can interact with each other via public methods or attributes
  4. Methods and attributes can be hidden from other classes by setting them as private
  5. The bundling of methods and attributes into objects with access control is called encapsulation

Object oriented code works on dividing up projects into objects, consisting of attributes and methods. Objects are instances of classes, and can interact with each other through public methods or attributes. Object oriented programming is good for modeling real life situations. The use of classes and methods allows developers to easily break down problems into much smaller and easier-to-solve chunks. You can encapsulate methods and attributes into objects with access control. Some of the basic definitions of OOP terms can be found below.

    Some key terms that are essential for OOP

  • Classes and Objects - Classes provide a template or blueprint for objects. Objects are an instance of a class.
  • Attributes - Attributes are properties or characteristics of objects, given in the form of variables
  • Methods - Methods are actions of objects, given in the form of procedures or subroutines
  • Constructor - The default method of a class that creates a new instance of an object
  • Access Modifiers - Determine whether parts of the program can be accessed outside their respective class (Public/Private)
  • Encapsulation - “Hiding” certain parts of the program from others. This prevents parts of the program outside the class from directly accessing it
  • Accessor Methods - Methods that pass private attributes of an object outside the class
  • Static vs Instance - Static methods are not linked to a specific object, but instead belong to the class. Instance methods are linked to an instance of a class (A specific object)
  • Value vs Reference - Value data types make a copy of the original data and pass that through. This means the original is not affected. By reference data types refer to the memory location of the data, meaning the value changes when updated.
  • Refactoring - Changing the layout or format of code to make it more efficient or tidier, but not changing the functionality

Recursion

Recursion is when a function calls itself

    Principles of recursion

  • Recursion can be used in place of a loop
  • There is usually an explicit stopping condition called the base case
  • The base case should be reached after a finite number of function calls
  • If a subroutine is called too many times, it can cause a stack overflow error
  • Systems handling recursion require substantial amounts of memory
  • However, recursive programs are often shorter than non-recursive equivalents

Object Oriented Programming - Inheritance

Inheritance allows classes to extend features and properties from others

    Key points about inheritance

  • Classes can acquire methods and attributes from other classes using inheritance
  • Subclasses inherit properties from superclasses
  • When methods are overridden a version of that method replaces the inherited method
  • Overloading is when different versions of a method with the same name exist to accept different parameters
  • Polymorphism describes different behaviour in different contexts due to overriding and overloading
  • Abstract classes cannot be instantiated, they represent ideas to be inherited by other classes

Types of Languages

Most languages can be sorted into one of the following paradigms

    Imperative paradigms

  • Assembly | x86, ARM, LMC | Low level languages with 1:1 correspondence between machine code and CPU instructions. Support variable names and labels
  • Procedural | C, Go | Structured code using sequence, selection, iteration and recursion. Code can be modular through use of procedures
  • Object Oriented | Java, C#, C++ | Make use of classes and objects which define behaviour. Follows the principles defined above
  • Scripting | JavaScript, Bash, Python | Languages designed for automation rather than creating full applications. These are often used in embedded systems

    • Declarative paradigms

  • Functional | Haskell, Groovy, Kotlin | Code is divided into functions. Functions are passed as arguments to higher order functions. Arguments and return values are used in place of global variables. Closely linked to maths
  • Logic | Prolog | The code is made from a ruleset which defines a scenario. Answers are obtained by asking specific questions
  • Data Query | SQL | Provide a way of interacting with data structures such as a database. These work by using queries to specify specific data rather than how to retrieve it.

Language Generations

Languages range in 'generations'; for example first generation refers to binary machine code. Second generation languages are architecture specific (Assembly). Third gen are high level (Procedural, Object Oriented, Scripting, Functional). Fourth generation languages are used for data manipulation, originally just SQL and other query languages, these features are now being implemented into high level languages, making the definitions less clear. Logic languages come under the definition of fifth generation, however these have very few applications today.
