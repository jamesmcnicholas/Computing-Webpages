Object Oriented Programming - Fundamentals

Many modern languages are based around OOP, including Java, C++, Python & more

The basics are as followed:

  1. Code is divided up into objects which contain variables (referred to as attributes) and functions (known as methods)
  2. Objects are made from "blueprints" called classes. Objects are an instance of a class
  3. Objects can interact with each other via public methods or attributes
  4. Methods and attributes can be hidden from other classes by setting them as private
  5. The bundling of methods and attributes into objects with access control is called encapsulation

Object oriented code works on dividing up projects into objects, consisting of attributes and methods. Objects are instances of classes, and can interact with each other through public methods or attributes. Object oriented programming is good for modeling real life situations. The use of classes and methods allows developers to easily break down problems into much smaller and easier-to-solve chunks. You can encapsulate methods and attributes into objects with access control. Some of the basic definitions of OOP terms can be found below.

  1. Classes and Objects - Classes provide a template or blueprint for objects. Objects are an instance of a class.
  2. Attributes - Attributes are properties or characteristics of objects, given in the form of variables
  3. Methods - Methods are actions of objects, given in the form of procedures or subroutines
  4. Constructor - The default method of a class that creates a new instance of an object
  5. Access Modifiers - Determine whether parts of the program can be accessed outside their respective class (Public/Private)
  6. Encapsulation - “Hiding” certain parts of the program from others. This prevents parts of the program outside the class from directly accessing it
  7. Accessor Methods - Methods that pass private attributes of an object outside the class
  8. Static vs Instance - Static methods are not linked to a specific object, but instead belong to the class. Instance methods are linked to an instance of a class (A specific object)
  9. Value vs Reference - Value data types make a copy of the original data and pass that through. This means the original is not affected. By reference data types refer to the memory location of the data, meaning the value changes when updated.
  10. Refactoring - Changing the layout or format of code to make it more efficient or tidier, but not changing the functionality

The Regulation of Investigatory Powers Act was passed in 2000.

The main purpose of this act is to provide restraints for security services and the police when investigating and surveying digital data and communications of individuals. The act also makes it a crime for anyone unauthorised by the act to carry out surveilance and monitoring of communicatons. The European Court of Human Rights stated that UK surveillance laws were unclear and that there was nothing in place to prevent abuses of power by organisations regarding the interception of electronic communications. It was decided that UK surveillance laws and practice must be tighter in order to protect the individual's rights to privacy. As a result, the act was passed.

The idea behind allowing these organisations to intercept and monitor comunications is to achieve the following:

  • Prevent or detect crimes
  • Prevent public disorder from occurring
  • To ensure national security and the safety of the general public
  • To investigate or detect any abnormal or illegal use of telecommunication systems.
